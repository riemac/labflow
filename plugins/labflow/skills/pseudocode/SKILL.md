# 框架与伪算法编写规范

本 skill 定义"算法草案阶段"的代码产出标准，最重要是思想和 idea 的准确传达。产出物是 **Python 骨架 + 注释级伪算法**——
既能被人审阅校正，又能直接作为 coding prompt 喂给下游模型完成实现。

## 核心定位

用户在此阶段对算法流程拥有**精确控制权**。Agent 的职责是：
- 将已经明确的算法共识忠实落地为代码骨架
- 补全注释级伪算法的数学表达和输入/输出约定
- 主动做交叉验证（公式 vs preset 数据）并记录偏差

Agent **不应**：
- 自行编写正式实现代码（`pass` / `raise NotImplementedError` 即可）
- 超出讨论范围添加字段、方法或抽象层
- 对用户已确认的设计决策做二次质疑

若任务还处在 idea 具现化、接口设计、字段语义打磨、模块职责切分阶段，优先使用 `design-scaffold`。

## 文件结构模式

一个典型的模块文件分为四层：

```
module_docstring          ← 设计说明（分类、职责边界、与其他模块的关系）
imports + constants       ← 模块级常量
Cfg dataclasses           ← 声明式配置类（字段 + 字段文档 + __post_init__ 校验）
Runtime classes           ← 运行时壳（核心方法只含 pass + TODO 伪算法注释）
```

### 模块文档字符串

放在文件最顶部，用 `r"""..."""`。内容侧重**设计决策和职责边界**，不是 API 参考：

```python
r"""模块的职责定位和覆盖范围。

分类说明（如有子类分化）：
- **子类 A**：...
- **子类 B**：...

设计说明
--------

### 关键设计决策
... 为什么这样分层、Preset 引用 / 职责边界 ...
"""
```

### 声明式配置类 (Cfg)

使用 `@dataclass`。每个字段必须有**字段文档字符串**（紧跟字段下方的 `"""..."""`）。

```python
@dataclass
class ExampleBuilderCfg(BuilderCfg):
    r"""一句话说明该 Cfg 的定位。

    补充说明字段设计原则、与其他 Cfg 的关系。
    """

    field_a: TypeA = default
    """字段含义。取值范围、单位、与其他字段的联动关系。"""

    field_b: TypeB = None
    """字段含义。为 None 时的语义。"""

    def __post_init__(self):
        # 校验逻辑：类型守卫、值域检查、联动推导
        ...
```

关键原则：
- 字段类型如有 union，在文档中注明各模式的语义差异
- `__post_init__` 负责校验和联动推导
- 默认值要有实际意义，不能为了"通用"而留 None

### 运行时壳

```python
class ExampleOperator(Operator):
    r"""一句话说明该运行时类的职责。"""

    cfg: ExampleOperatorCfg

    def __init__(self, cfg: ExampleOperatorCfg):
        self.cfg = cfg

    def run(self) -> OutputCfg:
        r"""核心方法（build / validate / export / generate 等）。"""
        pass

        # TODO:算法之一（名称）
        # ...伪代码...
```

核心方法内只保留 `pass` + TODO 注释块，不写实现。

## TODO 伪算法块格式

这是本 skill 的核心产出格式。每个 TODO 块是一段自包含的算法规格。

### 结构

```python
# TODO:算法之X（算法名称）
# ────────────────────────────────────────
# 输入
#   参数说明...
#   可调参数（均提供默认值，用户可覆盖）：
#     $param$ — 含义，默认值
#
# 输出：类型/结构说明
#
# ── 计算步骤标题 ──
#   公式或规则...
#
# ── 与 preset 的交叉验证 ──
#   公式值 vs 实测值，偏差分析
#
# IDEA：设计洞察或后续迭代思路
```

### IDEA 注记

用于记录设计洞察、工作流预期、与后续模块的衔接点。

## 视觉分隔

模块内的大段落用分隔线：

```python
# ============================================================================
#  段落标题（如"整手的声明式配置类"）
# ============================================================================
```

TODO 块内用较轻的分隔：

```python
# ────────────────────────────────────────
```
