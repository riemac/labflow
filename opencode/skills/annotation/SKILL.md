# 注释规范

## 核心定位：你在为科研人员写代码，不是在写软件工程项目

**使用中文。**

这份规范的读者是**会逐行精读代码、需要核对建模意图的科研人员**，不是只看接口签名的软件工程师。
注释的首要目的是"让读者在不离开代码文件的情况下，完成实现与建模意图的完整对照"。

若开发环境支持 VSCode LaTeX 注释渲染，注释应被视为**代码内嵌的建模文档**，而不是装饰性文字。

---

## ⛔ 强制禁止行为（违反即视为输出错误，无任何例外）

### A. 注释密度禁令

1. **禁止输出注释稀疏的代码**：注释行（含文档字符串行、行间注释行、含行右注释的代码行）占文件总行数的比例**不得低于 67%**，科研核心算法文件的目标密度为 **90%**。
2. **禁止省略行间注释**：每个非平凡代码块（超过 2 行的逻辑单元）上方必须有行间注释，说明这一段的职责和动机。
3. **禁止省略行右注释**：凡关键赋值、数值操作、公式项、形状/维度变换，必须有行右注释，说明物理/数学含义、量纲或数据形状。
4. **禁止省略文档字符串**：每个公开函数、类、方法都必须有文档字符串，不得以"功能显而易见"为由省略。

### B. 科研内容禁令

5. **禁止删除 LaTeX 算法注释**：用户在注释中写的公式、算法输入输出规格、数学推导，是建模规格文档，**不是临时脚手架**，必须完整保留并迁移到实现代码的对应位置。
6. **禁止用软件工程思维替换科研思维**：不得把"算法输入 $\mathbf{x} \in \mathbb{R}^{N \times d}$，学习率 $\eta$"这类建模规格注释替换为"初始化张量"、"设置参数"这类软工描述。
7. **禁止删除数值锚点**：用户标注的 preset / 超参 / 实验数值是算法正确性的参照基准，必须保留在对应类/函数的文档字符串或行间注释中。
8. **禁止删除背景知识注释**：领域背景说明、坐标系/符号约定、与论文/图示/数据集的对应关系、设计动机分析，均属于代码的组成部分，不得以"过于冗长"为由删减。

### C. 特殊标记规则

9. **禁止丢失以下标记后的科研信息量**，可以重新组织措辞、迁移位置，或随实现状态更新标记，但信息量不得减少：
   - `# TODO:` — 待实现规格，通常来自 scaffold 阶段，含算法规格、字段语义、边界或验收条件
   - `# DONE:` — AI 已实现并等待用户确认的中间状态，仍需保留原规格语义
   - `# NOTE:` — 重要设计决策或约定
   - `# IDEA:` — 设计备忘
   - `# Question:` — 未决问题
   - `# preset:` / `# preset：` — 数值锚点
   - `# ref:` — 文献/图示引用

> **自检标准**：生成代码后，逐项过一遍以上禁令。若任意一项被违反，重新生成。

## 1. 注释密度目标

| 文件类型 | 最低注释行占比 | 目标注释行占比 |
|---|---|---|
| 科研核心算法（建模、数值计算、物理/几何变换） | 80% | **90%** |
| 配置类 / 数据类 / 超参类 | 60% | 80% |
| 工具函数 / 解析函数 / 预处理 | 60% | 80% |
| 纯 schema / 注册表 / 常量定义 | 60% | 80% |

**注释行的计算定义**：文档字符串行、独立 `#` 注释行、含行右 `#` 注释的代码行，均计入注释行。空行不计入分母。

**达到目标密度的标准路径**：
- 每个变量初始化都有行右注释（含量纲、形状、物理含义）
- 每个逻辑块上方都有行间注释（含动机和与算法公式的对应关系）
- 每个函数/类都有完整文档字符串（含公式、数值锚点、文献引用）
- 每个关键公式项都有 LaTeX 行右注释

## 2. 文档字符串规范

采用 Google Doc String 风格，使用原始字符串 `r"""..."""`。

### 函数文档字符串模板

```python
def function_name(param1: Type1, param2: Type2) -> ReturnType:
    r"""函数功能的简要描述。

    详细解释该函数解决的问题、背后的原理或动机。
    与论文/图示的对应关系（如"对应论文 Algorithm 1 第 3 步"）。

    核心公式：
    $$
    \mathbf{y} = f(\mathbf{x};\,\theta) = \sigma(\mathbf{W}\mathbf{x} + \mathbf{b})
    $$

    数值锚点（preset / 超参 / 实验配置，作为正确性参照保留）：
    $d=256,\ \eta=3\times10^{-4},\ \beta=(0.9,\,0.999)$

    Args:
        param1 (Type1): 参数含义，说明取值范围或量纲。
        param2 (Type2): 参数含义。

    Returns:
        ReturnType: 返回值含义和形状（如适用）。

    Raises:
        ExceptionType: 何时抛出此异常。
    """
```

### 类文档字符串模板

```python
class MyClass:
    r"""类的简要描述。

    详细说明类的职责、在系统中的角色、设计考量。

    领域背景、符号约定、与论文/数据集/外部文件的对应关系等，
    完整保留于此，不得以"过于冗长"为由删减——
    这些是科研人员核对建模意图的依据。

    符号约定：$N$ 为批大小，$d$ 为特征维度，$T$ 为时间步总数。
    """
```

## 3. 行内注释规范

### 行间注释（代码块上方）

说明这一段在做什么、为什么这样做、与算法公式的对应关系：

```python
# 计算归一化因子 $Z = \sum_j \exp(q_k^\top k_j / \sqrt{d})$，防止 softmax 数值溢出
# 这里除以 $\sqrt{d}$ 是论文 Eq.(1) 的缩放项，不是工程近似
scores = (query @ key.transpose(-2, -1)) / math.sqrt(d)
```

### 行右注释（关键赋值右侧,饱满注释的关键，且不占额外行数，重要程度极高）

说明该行的物理/数学含义、数据形状、量纲：

```python
x = torch.zeros(B, T, d)          # 输入序列，形状 [B, T, d]，初始化为零
scale = 1.0 / math.sqrt(d_k)      # 缩放因子 $1/\sqrt{d_k}$，防止点积过大导致梯度消失
loss = F.cross_entropy(logits, y)  # 交叉熵损失 $\mathcal{L}=-\log p(y|\mathbf{x})$，标量
```

### LaTeX 公式注释

凡代码实现了一个数学公式，必须在行右或行间写出对应的 LaTeX：

```python
# 指数移动平均 $\hat{m}_t = m_t / (1 - \beta_1^t)$，修正初始化偏差
m_hat = m / (1 - beta1 ** t)   # $\hat{m}_t$，偏差修正后的一阶矩估计
```

## 4. 注释迁移规则（草稿 → 实现时强制执行）

用户草稿代码中的注释是**算法规格文档**，不是临时脚手架。
实现时的原则：**在草稿注释的基础上"填肉"，草稿注释是骨架，不能拆。**
允许重新组织注释的位置和措辞，但**信息量只能增加，不能减少**。

`TODO:` 是设计脚手架阶段留给实现阶段的规格锚点，不是永久不可改的标签。
实现完成时，应把对应 `TODO:` 更新为 `DONE:` 或迁移为普通科研注释，并保留原有公式、字段说明、边界条件和验收语义。
部分完成时，只把已兑现部分改为 `DONE:` 或普通科研注释；未完成或待用户判断的部分继续保留 `TODO:` / `Question:`。
`DONE:` 只表示 AI 已实现并等待用户确认；最终清理由 git 收尾阶段处理，用户确认前不得删除其承载的科研语义。

| 草稿中的内容 | 实现后的位置 | 处置方式 |
|---|---|---|
| 文件头部背景说明（领域知识、符号约定、模型假设） | 文件头部模块文档字符串 | **完整保留**，不得截断 |
| `# TODO:` 算法输入输出规格（含 LaTeX 公式） | 对应函数文档字符串 + 实现代码的行间注释 | 已实现则改为 `# DONE:` 或普通科研注释；未实现则继续保留 `# TODO:` |
| 数值锚点（preset / 超参 / 实验配置） | 对应类/函数文档字符串 | **完整保留** |
| `# NOTE:` 设计决策 | 对应代码段上方行间注释 | **完整保留** |
| `# Question:` 未决问题 | 对应位置 | **完整保留** |
| `# IDEA:` 备忘 | 对应位置 | **完整保留** |
| 文献/图示引用 | 对应文档字符串或行间注释 | **完整保留** |
| 与外部文件/数据集/URDF/配置的对应关系 | 对应文档字符串 | **完整保留** |
| 符号/坐标系/维度约定 | 对应类/函数文档字符串 | **完整保留** |

## 5. 数学公式规范

- 行内公式：`$...$`
- 独立公式块：`$$...$$`
- 变量斜体：`$x$`、`$\theta$`、`$l_i$`
- 矩阵/向量粗体：`$\mathbf{W}$`、`$\mathbf{x}$`
- 函数正体：`$\sin$`、`$\exp$`、`$\text{softmax}$`
- 集合/空间：`$\mathbb{R}^{N \times d}$`、`$\mathcal{L}$`
- **禁止**使用 `:math:\`...\`` 写法

**公式使用时机**：
- 有明确数学建模、数值变换、概率定义、几何变换时，**主动写公式**
- 代码实现的是理论公式的工程近似版时，必须同时写出：理论公式 + 工程改动 + 改动原因（数值稳定 / 接口兼容 / 效率）
- 纯 schema / 字段解析 / 路径拼接类代码，无需强行写公式

## 6. 优先解释的内容（科研场景专项）

以下内容在科研代码中**必须注释**，不得遗漏：

- 张量/数组的形状、量纲、数值范围、边界条件
- 算法中每个公式项的物理/统计意义（不只是变量名，而是"这代表什么"）
- 数值锚点的来源（"来自论文 Table 2"、"消融实验最优配置"等）
- 设计取舍（"这里用 LayerNorm 而不是 BatchNorm，是因为……"）
- 工程近似与理论模型的偏差（"这里用 clamp 替代了理论上的投影，原因是……"）
- 超参/常数的物理或统计意义（"$\beta_1=0.9$ 对应约 10 步的指数衰减窗口"）
- 与其他模块/变体/基线的差异点（"与标准 Transformer 相比，这里去掉了位置编码"）

## 7. 完整示例

### 示例 A：科研核心算法函数（含公式迁移与行右注释）

```python
def scaled_dot_product_attention(
    query: torch.Tensor,
    key: torch.Tensor,
    value: torch.Tensor,
    mask: torch.Tensor | None = None,
) -> torch.Tensor:
    r"""计算缩放点积注意力（Scaled Dot-Product Attention）。

    对应 Vaswani et al. (2017) Eq.(1)：
    $$
    \text{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V})
      = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}
    $$

    除以 $\sqrt{d_k}$ 的动机：点积的方差随 $d_k$ 线性增长，缩放后
    使 softmax 输入落在梯度较大的区域，避免饱和。
    这是理论上的精确缩放，不是工程近似。

    Args:
        query (torch.Tensor): 查询矩阵，形状 $[B, H, T_q, d_k]$。
        key (torch.Tensor): 键矩阵，形状 $[B, H, T_k, d_k]$。
        value (torch.Tensor): 值矩阵，形状 $[B, H, T_k, d_v]$。
        mask (torch.Tensor | None): 可选因果掩码，形状 $[1, 1, T_q, T_k]$，
            被遮蔽位置填 $-\infty$ 使 softmax 后权重趋近于 0。

    Returns:
        torch.Tensor: 注意力输出，形状 $[B, H, T_q, d_v]$。
    """
    d_k = query.size(-1)  # 键/查询的特征维度 $d_k$，用于计算缩放因子

    # 计算原始注意力分数 $\mathbf{Q}\mathbf{K}^\top / \sqrt{d_k}$，形状 [B, H, T_q, T_k]
    # matmul 对最后两维做矩阵乘，前两维自动广播
    scores = (query @ key.transpose(-2, -1)) / math.sqrt(d_k)  # $\mathbf{Q}\mathbf{K}^\top/\sqrt{d_k}$

    # 应用因果掩码：把未来位置的分数置为 $-\infty$，softmax 后权重趋近于 0
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))  # 遮蔽位置 → $-\infty$

    # softmax 归一化得到注意力权重，沿 $T_k$ 维归一化，形状 [B, H, T_q, T_k]
    weights = F.softmax(scores, dim=-1)  # $\alpha_{ij} = \exp(s_{ij})/\sum_j \exp(s_{ij})$

    # 加权求和得到输出，形状 [B, H, T_q, d_v]
    return weights @ value  # $\sum_j \alpha_{ij} \mathbf{v}_j$
```

### 示例 B：配置类（含数值锚点与设计取舍注释）

```python
@dataclass
class AdamWConfig:
    r"""AdamW 优化器超参配置。

    对应 Loshchilov & Hutter (2019) 的解耦权重衰减版本。
    与标准 Adam 的区别：weight_decay 作用于参数本身而非梯度，
    使正则化强度与学习率解耦，在 Transformer 训练中表现更稳定。

    论文推荐默认值（GPT-2 规模实验最优配置）：
    $\eta=3\times10^{-4},\ \beta=(0.9,\,0.95),\ \epsilon=10^{-8},\ \lambda=0.1$

    NOTE: $\beta_2=0.95$ 而非 Adam 默认的 $0.999$，
    是因为语言模型梯度噪声较大，较小的 $\beta_2$ 使二阶矩估计衰减更快，
    对梯度突变的响应更灵敏。
    """

    lr: float = 3e-4           # 学习率 $\eta$，控制参数更新步长
    betas: tuple[float, float] = (0.9, 0.95)  # 一/二阶矩指数衰减系数 $(\beta_1, \beta_2)$
    eps: float = 1e-8          # 数值稳定项 $\epsilon$，防止分母为零
    weight_decay: float = 0.1  # 解耦权重衰减系数 $\lambda$，仅作用于非偏置参数
```
